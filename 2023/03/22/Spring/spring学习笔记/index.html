spring学习笔记

一、pring的IoC和DI

1、Spring简介

1.1 Spring是什么

  • Spring是分层的 Java SE/EE应用 full-stack 轻量级开源框架，以 IoC（Inverse Of Control：反转控制）和 AOP（Aspect Oriented Programming：面向切面编程）为内核。

  • 提供了展现层 SpringMVC 和持久层 Spring JDBCTemplate 以及业务层事务管理等众多的企业级应用技术

2.2 优势

  • 解耦，简化开发。

    • 提供了IoC控制反转容器。可以将对象间的依赖关系交由 Spring 进行控制，不用直接new，直接找spring容器。避免硬编码所造成的过度耦合。 用户也不必再为单例模式类、属性文件解析等这些很底层的需求编写代码，可以更专注于上层的应用。

  • AOP 编程的支持

    • 通过 Spring的 AOP 功能，方便进行面向切面编程，许多不容易用传统 OOP 实现的功能可以通过 AOP 轻松实现。

  • 声明式事务的支持

    • 编程式控制事务写代码重复，通过声明式事务的控制可以简化，一次性，自动控制，方便提高开发效率

  • 方便程序的测试

    • 可以非容器依赖的编程方式进行几乎所有的测试工作

  • 集成优秀的框架

    • Spring对各种优秀框架（Struts、Hibernate、Hessian、Quartz等）的支持。web层有springMVC，在dao层有jdbc模板，

  • 降低 JavaEE API 的使用难度

    • 进行了工具的封装，使用这些API的难度降低

2.3 体系结构

test：所有框架都可以进行测试

core container：核心容器，包括四个部分，之前都要导入四个jar包，现在导入maven坐标即可。

再上一层：

data access/integration: 数据访问层

web：网络层

2. Spring快速入门

2.1 程序开发步骤

  • 在业务层调用dao层。即在业务层的UserServiceImpl调用dao.UserDaoImpl对象，调用方式new对象，用接口去接收具体UserDaoImpl的实现。
  • 需要完成解耦合，需要创建xml配置文件。需要UserDaoImpl全限定名配置到配置文件的内部。通过一个id标识当前这个全限定名:通过这个标志去获得全包名。
  • 引入spring框架，读取配置文件，从而获得全包名获得对象。
  • 通过spring框架，就不需要通过接口new对象了。现在通过spring要对象，即现在getBean方法，然后走到spring框架
  • spring框架需要做两件事，读xml文件；通过id表示，获得全限定名。框架通过反射创建对象，并返回对象

导入spring对应的maven坐标

需要编写dao接口和UserDaoImpl实现类

创建Spring核心xml配置文件

在 Spring 配置文件中配置 UserDaoImpl

使用 Spring的客户端的getBean方法传入对象标识，最终返回指定对象，找spring要对象，而不是new对象。 的 API 获得 Bean 实例

2.2 导入Spring开发的基本包坐标

  • 打开idea，创建一个空的Spring项目，然后创建一个基于web的maven模块itheima_spring_ioc模块。

  • 在pom.xml文件中，导入Spring开发的基本包坐标

2.3 编写Dao接口和实现类

  • 在java包下，创建一个com.itheima.dao包，在该包下创建一个UserDao接口

  • 然后在com.itheima.impl包下创建一个UserDaoImpl的实现类，实现这个接口，在里面，简单写一个save方法

2.4 创建Spring核心配置文件, 并且配置UserDaoImpl

  • 在类路径下，resources包下，创建applicationContext.xml配置文件，这个名字约定俗成。在该包下右键。new -> xml configuration file -> Spring config => applicationContext 回车

2.5 使用Spring的API获得Bean实例

  • 在com.itheima.demo包下创建一个UserDaoDemo类，编写main方法，使用这个API获得Bean实例对象，这里需要做强转。然后使用这个对象调用save方法。（创建ApplicationContext对象getBean）

3. Spring配置文件

3.1 Bean标签基本配置（在applicationContext.xml里面配置）

  • id不允许重复，唯一性标识。后面是全限定名，默认反射通过无参构造创建对象，所以需要保证bean内部有无参构造

  • scope：指对象的作用范围，取值如下：

    取值范围 说明
    singleton 默认值，单例的，意味着在容器中存在的对象只有一个，地址一样
    prototype 多例的，意味着在容器中有多个对象，每次getBean时都会获得一个新的对象
    request WEB 项目中，Spring 创建一个 Bean 的对象，将对象存入到 request 域中
    session WEB 项目中，Spring 创建一个 Bean 的对象，将对象存入到 session 域中
    global session WEB 项目中，应用在 Portlet 环境，如果没有 Portlet 环境那么globalSession 相当 于 session

  • xml配置

  • 将scope参数设置为singleton时，说明容器中存在的对象只有一个，当打印多个对象的时，输出的地址会一样。在src目录下，创建一个test目录，在该目录下创建对应的java目录，然后创建，com.itheima.test包，在该包下创建一个SpringTest测试类。

  • 当scope参数设置为prototype时，使用同样经过上面的操作，会使得多个对象打印出的地址不一样

总结：

当scope的取值为singleton时。

  • Bean的实例化个数：1个；
  • Bean的实例化时机：当Spring核心文件被加载时，实例化配置的Bean实例。
  • Bean的生命周期：
    • 对象创建：当应用加载，创建容器时，对象就被创建了
    • 对象运行：只要容器在，对象一直活着
    • 对象销毁：当应用卸载，销毁容器时，对象就被销毁了

当scope的取值为prototype时。

  • Bean的实例化个数：多个；
  • Bean的实例化时机：当调用getBean()方法时实例化Bean
    • 对象创建：当使用对象时，创建新的对象实例
    • 对象运行：只要对象在使用中，就一直活着
    • 对象销毁：当对象长时间不用时，被 Java 的垃圾回收器回收了

3.3 Bean生命周期配置

  • init-method：指定类中的初始化方法名称

  • destroy-method：指定类中销毁方法名称

  • 在UserDaoImpl类中，写一个无参构造方法，写一个初始化init方法和一个销毁destroy方法。然后在配置里面applicationContext.xml里面的bean属性中，加入init-method和destroy-method分别调用我们写的初始化方法和销毁方法。

3.4 Bean实例化三种方式

无参构造方法实例化 (重点)

  • 上面已经讲到了，之前都是利用的无参构造来实例化的

工厂静态方法实例化 (了解)

  • 首先写一个静态工厂类，在com.itheima.factory包下创建该类，简单写一个返回一个实例对象的方法。

  • 然后，在xml文件中，配置bean中class属性和工厂方法属性。

工厂实例方法实例化 (了解)

  • 写一个动态工厂类，简单写一个返回实例对象的方法

  • 在xml文件中，配置方法属性

Spring Boot打成的 jar 和普通的jar有什么区别呢

Spring oot 项目最终打包成的 jar 是可执行 jar ，这种 jar 可以直接通过 java -jar xxx.jar 命令来运行

普通打包会缺少能让项目启动的配置jar包，不能够直接运行。

Spring Boot的核心注解是哪些？他由哪几个注解组成的

核心注解是@SpringbootApplication，包含了一下三个注解：

@SpringbootConfiguration：组合了Configuration注解，实现配置文件的功能

@EnableAutoConfiguration：打开自动配置的功能

@ConponentScan：spring组件扫描

spring-boot-starter-parent 有什么用?

每一个springboot项目都有一个parent标签，作用如下：

  1. 统一定义配置。定义java版本、定义编码方式等
  2. 统一管理依赖和版本。依赖管理标签中定义了非常多的指定版本依赖，方便在我们导入依赖坐标是不需要指定版本，避免版本冲突

什么是Redis？简述它的优缺点

redis本质上是一个key-value类型的内存数据库，把数据加载到内存，操作在内存中进行

优点：操作在内存进行，所以处理速度非常快，而且支持保存多种数据结构

缺点：容易收到内存限制，不能用于海量数据的高性能读写

Redis有哪些数据结构(类型)

redis有五种基础数据结构

  • string 普通字符串，常用
  • list 按照插入顺序排序，可以有重复元素 适合任务队列
  • hash 适合存储对象
  • set 无序集合，没有重复元素
  • zset 有序结合，没有重复元素，适合排行榜

redis是单线程还是多线程，为什么这么快

redis在6.0版本之前是单线程，在这之后是多线程。在单线程下省去了线程切换，和锁竞争的开销，比较快

更重要的是它基于内存操作

redis如何实现分布式锁

为什么redis要将所有数据放到内存中

是为了达到最快的读写速度而放在内存中，并且通过异步的方式写入到磁盘中

如果不放在内存中，磁盘io速度会影响redis性能

缓存失效、缓存穿透、缓存雪崩、缓存并发

缓存雪崩

  • 定义：大量的key设置相同的过期时间，导致缓存在同一时间全部失效，导致瞬时数据库请求量大
  • 解决办法：给缓存的过期时间加上一个随机值时间，将过期时间分散开

缓存穿透

  • 定义：查询一个数据库不存在的数据，由于缓存的机制，每次查询都会从数据库找数据，然而数据库没有，这样的访问会失去缓存的意义
  • 解决办法
    • 可以在封装get和set时增加一个步骤，如果查询的key不存在，就以这个key为前缀设定一个表示key，等下一次查询该key时，先查询标识key，如果该key存在，就返回协定好的非false或者null值
    • 当返回结果是空时，仍然把这个空结果进行缓存里面
    • 采用布隆过滤器
    • 接口层增加校验，加强用户权限校验

缓存击穿

  • 定义：一个存在的key，在缓存过期的一刻，大量请求会打入数据库，造成压力
  • 解决办法：在访问key之前，采用setnx来设置另一个短期key来锁住当前key的访问，访问结束就删除当前短期的key

缓存失效

  • 定义：是指一个缓存在高并发下，当一个缓存在某一时间失效时，在高并发下，导致数据库的瞬间压力过大
  • 解决办法：可以将缓存的失效时间分散开

缓存并发

  • 定义：高并发下，如果一个缓存失效，需要同时设置缓存，如果并发量过高，也能导致数据库压力过大
  • 解决办法：对缓存加锁，如果不存在，就加锁，然后查询数据库加入缓存，然后解锁。其他进程如果发现有锁时，就等待其解锁

Redis中的热key怎么处理

缓存中的某些key对应的value存储在集群中的一台机器上，使得所有流量涌向同一台机器，限制性能

将热点key对应的value存储在客户端本地，并设置一个失效时间

将热点key分散为多个子key，存储在不同机器上。当通过热点key去查询数据时，通过某种hash算法随机选择一个子key，然后在访问对应缓存机器
